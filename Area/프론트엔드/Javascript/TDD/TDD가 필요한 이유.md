
### 잘못된 코드를 작성하기 쉽다
```js
console.log = 1000
console.log() // type 에러발생 
```

이게 가능하다
자바와 같이 빌드과정이 있는 언어는 문법검사를 통해 프로그램 실행 전 에러를 방지할 수 있음
하지만 자바스크립트는 빌드 과정이 없이 실행된다. 코드를 작성하고 브라우저에서 실행해봐야 에러를 확인할 수 있음

### 타입이 헷갈린다
```js
'1' + 1 = ? // '11'
'2' * 3 = ? // 6
1 + '2' + 3 * 4 = ? // '1212'
```

### 컴파일러가 없으니 테스트가 최선이다
이러한 문제를 극복하기 위해 테스트 주도 개발(TDD) 을 사용함
