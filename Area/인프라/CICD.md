- **Continuous Integration** : 
	1. 코드 변경사항을 주기적으로 빈번하게 머지해야 한다.
	2. 통합을 위한 단계(빌드, 테스트, 머지)의 자동화
		- 코드 리뷰
		- 머지
		- 빌드
		- 유닛 테스트, Integration 테스트 등
		- 문제를 일으킨 커밋에 알리기
		- 이후 배포과정
- **Continuous Deployment** :  
	1. 릴리즈할 준비과정
	2. 검증(수동) 혹은 자동으로 사용자에게 배포

- **Tools** : 
	- Jenkins
	- Buildkite
	- GitHub Actions
	- GitLab CI/CD
	- Bitbucket Pipelines
	- circleci
